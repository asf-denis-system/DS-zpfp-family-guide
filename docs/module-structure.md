# Структура модулей: «Нулевые и первые принципы для семьи»

> Ответ на запрос Церена: модули (разделы), темы (подразделы), содержание, стиль выдачи.
> Это новая программа в архитектуре WP-149, рядом с СС/ПС/ВСМ/IWE.

---

## Архитектура программы

**Аудитория:** Взрослый, у которого есть регулярный доступ к ребёнку 3-10 лет (родитель, бабушка/дедушка, старший родственник, воспитатель, тьютор). На начальном этапе — из сообщества МИМ.
**Пререквизит:** Минимальный — Личное развитие (ритм, заметки, экзокортекс). Идеальный — R1-R4 рабочего развития.
**Формат:** Бот выдаёт контент фрагментами. Адаптирует под: возрастную зону ребёнка, прогресс наставника в МИМ, контекст семьи.
**Длительность:** Непрерывная (цикл на принцип ~1 мес, 8 принципов = ~8 мес первый круг).

**Три возрастные зоны** (стартовая точка по возрасту, далее корректируется по наблюдениям):

| Зона | Возраст | Суть | Длительность эпизода | Форма ответа ребёнка |
|------|---------|------|---------------------|---------------------|
| **Младшие** | 3-5 | Действие, жест, 1 слово | 3-7 мин | «Покажи», «выбери из двух» |
| **Старшие дошкольники** | 5-7 | Фраза, начало «почему», простой критерий | 7-12 мин | «Скажи почему», «назови правило» |
| **Школьники** | 7-10 | Объяснение, версия-проверка, запись | 10-20 мин | «Две версии», «как проверим», «запиши» |

> Границы нежёсткие: бот сдвигает зону по фактическому уровню ребёнка (из Mastery Map и наблюдений наставника). Возраст — стартовая точка, дальше — по данным.

**Два контура:**
1. **Контур взрослого** — сам учится (принципы, методика, роль наставника) и параллельно ведёт ребёнка. Глубина и самостоятельность зависят от прогресса в МИМ:

| Этап взрослого в МИМ | Что может сам | Где нужна поддержка бота |
|----------------------|---------------|--------------------------|
| **Ещё не начал** (ниже L1) | Следует готовой стратегии, ведёт эпизоды по карточкам | Стратегия, программа, выбор принципа, Guard Policy — всё задаёт бот |
| **Личное развитие** (L1) | + проектировать ритм и среду, вести лог, адаптировать стратегию | Генерация контекстов, FM-detection |
| **R1-R4** (рациональная работа) | + расширять контексты, связывать с предметами, участвовать в стратегировании | Корректировка по факту, сложные решения |
| **R5-R6** (системное мышление) | + добавлять FP-маркеры, понимать глубину принципов | Сложные различения |
| **R7+** (продвинутые методологии) | + видеть системы, расширять в проектную деятельность | Минимальная — бот как автоматизация рутины |

> Бот определяет уровень взрослого при онбординге и адаптирует: объём объяснений, степень самостоятельности, глубину промптов.

2. **Контур ребёнка** — получает мини-эпизоды через взрослого (зона и трек определяются по возрасту + наблюдениям)

## Модуль 3. Принципы: 8 ZP для детей

> Ядро программы. Каждый принцип — отдельный подмодуль (~1 месяц практики).
> Рекомендуемый порядок: Z2 → Z0 → Z3 → Z1 → Z5 → Z4 → Z6 → Z7.
> Бот выдаёт принцип, который сейчас в фокусе по стратегии ребёнка.

Для каждого принципа Z0-Z7 — одинаковая структура подмодуля:

| Тема | Содержание |
|------|-----------|
| 3.N.1 Что это (для взрослого) | Принцип: детский вопрос, различение, связь с ZP. Что наставник должен понимать |
| 3.N.2 Как выглядит по зонам | Для каждой из 3 зон (младшие 3-5 / старшие дошк. 5-7 / школьники 7-10): can-do, типичные ошибки, домены, формы подачи, FP-маркеры (для школьников) |
| 3.N.3 Банк эпизодов | 6-10 готовых карточек × 3 зоны × Трек A/B. Фразы взрослого, строка лога. Бот выбирает карточку под зону и трек |
| 3.N.5 Guard Policy | Когда переходить к следующему принципу: Level 3 (спонтанный перенос) в ≥2 контекстах, ≥3 раз |
| 3.N.6 Типичные поломки | FM этого принципа + как починить |

## Стиль выдачи текста (промпты)

> Ниже — наше предложение по стилю. Перед финализацией сверить с форматом выдачи существующих программ AIST (СС/ПС/ВСМ/IWE) и с техническими возможностями WP-149: потестировать @aist_me_bot, посмотреть шаблоны контента в aisystant/aist_bot, уточнить у Церена.

### Общие правила

1. **Адресат — взрослый-наставник**, не ребёнок. Тон: уважительный, инженерный, без морализаторства
2. **Короткие блоки** (1-3 абзаца), затем вопрос на понимание или действие
3. **Принцип «термин не раньше опыта»**: сначала практический пример, потом название
4. **Ссылки на руководства МИМ**: «Подробнее — в руководстве [название], §N»
5. **Адаптация под контекст семьи**: возраст ребёнка, уровень наставника в МИМ, ограничения по времени

### Типы выдачи

| Тип | Когда | Формат |
|-----|-------|--------|
| **Теория** (1.x, 2.x) | Наставник осваивает метод | Текст 200-400 слов → 2-3 вопроса на понимание → мини-задание письмом |
| **Принцип** (3.x) | Новый ZP-принцип в фокусе | Детский вопрос → объяснение для взрослого → 2 примера из быта → «попробуйте сегодня» |
| **Скрипт эпизода** (3.N.4) | Наставник готовится к эпизоду | Карточка: подготовка → что делать (Трек A/B) → фразы → лог. Для со-наставника — упрощённая версия |
| **Планирование** (5.x) | Начало недели/месяца/цикла | Бот генерирует план на основе: контекст семьи + текущий принцип + прогресс. Наставник утверждает |
| **Обратная связь** (4.x) | После цикла/недели | Бот анализирует лог → показывает прогресс по Mastery Map → предлагает корректировку |
| **FM-alert** | Бот детектирует failure mode | Мягкое предупреждение + конкретное действие: «Похоже на FM-01 (карго-культ): много эпизодов, мало переноса. Предлагаю: снизить частоту, добавить бытовой контекст» |

### Пример промпта для генерации скрипта эпизода

```
Ты — помощник наставника в программе «Нулевые и первые принципы для семьи».

Контекст семьи: {family_context}
Ребёнок: {child_name}, {age} лет, трек {preschool/school}
Текущий принцип: {current_principle} ({principle_question})
Текущий уровень: {mastery_level} (0-3)
Предыдущие эпизоды: {recent_episodes_summary}

Сгенерируй скрипт эпизода для наставника:
1. Подготовка (что нужно, 1-2 предмета из быта)
2. Трек A (базовый): шаги + фразы взрослого
3. Трек B (усложнение): что добавить если ребёнок бодрый
4. Строка лога (шаблон)
5. На что обратить внимание (типичные ошибки этого принципа)

Стиль: короткий, практичный, без лекций. Фразы взрослого — готовые к произнесению.
```
